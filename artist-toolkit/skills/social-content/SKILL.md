# Artist Social Content

The method for writing social content for an independent music artist. This skill carries
the *how*. The artist's identity, voice, positioning, and current release context come from
the active project — read them, don't restate them.

## Principle — method, not identity

This skill is artist-agnostic on purpose, so it can serve any music project. It never
hardcodes one artist's universe. Before producing anything:

- Pull the artist's voice, positioning, and identity markers from the active project's
  instructions and knowledge base.
- If no artist voice is defined in the project, ask for it before writing content.
- Never default to generic music-marketing hype. Match the artist's actual register.

## Memory of what works

Before generating, read the content tracker designated in the active project's instructions
(a Notion base or any other linked source, if one is designated) for hooks, captions, and CTAs
that already performed well, and for angles already used. Reuse what works; avoid repeating
what's been done. If no tracker is designated, ask which one to use. After the user picks
winners, they log them back to the tracker themselves — the skill stores nothing.

### What "performed well" means

Don't judge a post by raw views or likes alone — view and reach counts mostly reflect how far
the algorithm pushed a post, not how good it was. Weight the signals that show people actually
responded:

- **Retention** (average watch time / completion) — the strongest sign a video held attention,
  and usually what earns distribution in the first place.
- **Saves and shares** — high-intent actions; a viewer who saves or shares is worth far more
  than a passive like.
- **Engagement rate relative to reach** — resonance among those actually reached, separating a
  post that connected from one that merely traveled.

A post can rack up views while scoring low on all three (it traveled but didn't land), or get
modest reach while scoring high (it landed but wasn't pushed) — the second is the better creative
template to reuse. If the tracker carries a single performance flag or score, treat it as a
shorthand entry point, not a verdict: read the underlying axes (retention, saves/shares,
engagement-per-reach) distinctly, because one colour collapses signals that carry different
lessons.

## Workflow — Social content

When the user brings a post to prepare, establish four things first. Ask only what isn't
already clear from the message:

1. **Nature** — teaser / single release / behind-the-scenes / event / narrative
2. **Format** — video / carousel / photo / story (ask for a description or an upload if
   visual analysis would help)
3. **Objective** — drive streams / engagement / mystery-intrigue / event promo
4. **Then produce** — 10 hooks, 10 CTAs, 10 captions, adapted to the format and objective,
   in the artist's voice.

Then iterate on feedback: refine tone and direction.

Content rules:
- Match the artist's voice from the project (defer to the project, never to a generic default).
- No over-explanation — leave room for curiosity.
- Surface identity markers when relevant.
- Hooks: short, scroll-stopping, one idea each.
- Stay coherent with the release's theme.

## Output

Deliver clean and copy-ready: group the 10 hooks, 10 CTAs, and 10 captions in three clearly
labeled blocks.

## Boundary — produce here, critique elsewhere

This skill *produces* content. It does not critique, score, or validate an existing proposition,
and it does not brainstorm storytelling angles in the abstract. When the user wants feedback on a
draft, a coherence check against the project universe, or storytelling directions to explore
before any copy exists, hand off to the **creative coach** skill — then come back here to write
the chosen direction.
